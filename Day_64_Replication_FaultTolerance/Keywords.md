Replication - Copying data across systems
Failover - Automatic switch to backup during failure
High Availability - System always accessible
Fault Tolerance - System continues working despite failures
Redundancy - Multiple components doing same job
Heartbeat - Regular health check signal
Leader-Follower - One node leads, others replicate
Recovery Point Objective (RPO) - Acceptable data loss in failure
Recovery Time Objective (RTO) - Acceptable downtime duration
